# 開発フロー(メモ)

## いいなと思った設定
- デフォルトブランチを`develop`にしておく。
- `.github/`にテンプレートを充実させることにより、PRの抜け漏れを防いでいる。
- PRごとにIssueを立てることはタスク管理しやすいと思った。

## prefixについて
コミットメッセージの`prefix`は以下のバリエーションがある。

| 用語　　　　　| 内容             |
|:--------------|:-----------------|
| Add           | 追加             |
| Fix           | バグ修正         |
| Change        | 仕様変更         |
| Update        | 機能修正         |
| Remove        | ファイル削除     |
| Disable       | コメントアウト   |

他にも色々あるので調べてみる。
